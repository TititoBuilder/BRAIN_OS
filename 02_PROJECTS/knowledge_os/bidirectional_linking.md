# Bidirectional Linking

## What It Is
A bidirectional link is a connection between two notes that is visible from both
ends. When note A links to note B, B automatically shows that A points to it. The
link works in both directions without you having to create it twice, so every
connection you make is discoverable from either side.

## How It Works
You create a link one way, writing a reference to another note, and the system
automatically records the reverse. The note you linked to now displays its
backlinks, the list of every note that points to it. This means you can stand on any
note and see not just what it points to, but everything that points to it, which is
often where the surprising value lives: you discover which ideas reference this one
without ever having tracked that manually. In a vault, this turns a set of one-way
references into a fully navigable graph where every edge is traversable both ways.

## Why It Matters
One-way links lose half the information: you know A relates to B, but standing on B
you would never know about A. Bidirectional links recover that, making the
connection structure complete and the graph genuinely navigable. This is what makes
a linked note system more than a pile of documents, the backlinks reveal context and
relationships you did not deliberately record, surfacing how ideas actually connect
across your knowledge. It is the mechanism that makes the Zettelkasten graph and your
vault's connections actually explorable.

## The Pattern
Make every link visible from both ends so the whole connection graph is navigable.
Backlinks turn one-way references into a two-way web and surface relationships you
never explicitly tracked.
