## Patterns Discovered

### Event-Driven vs Scheduled Architecture Pattern
The BDF Knowledge System only performs work when files land in incoming\ folder - zero resource consumption when idle. This scales efficiently compared to scheduled polling, especially for sporadic knowledge compilation events.

### Pre-Routing Beats Post-Detection
Determining content routing during compile session response is more reliable than auto-detecting after file creation. The routing map step prevents misrouting issues that plagued early archive processing.

### PowerShell Profile Synchronization Pattern
Both Microsoft.VSCode_profile.ps1 and Microsoft.PowerShell_profile.ps1 must contain identical functions to ensure aliases work in all terminal contexts. VS Code terminals load the VSCode profile, standalone PowerShell loads the PowerShell profile.

### Cost Accumulation Pattern in Batch Processing
When multiple files touch the same chapter, each successive merge reads the output of all previous merges as input. Third merge pays for first two merges plus new content - tokens compound exponentially.

### Signal Leak in VS Code Integrated Terminals
Active Claude Code sessions leak KeyboardInterrupt signals to sibling terminal tabs within VS Code's integrated terminal multiplexer. Close Claude Code with /exit before running Python in adjacent terminals.

## Principles Earned

### One Input, One Run
Processing one compile file per bdf-book execution prevents cost compounding. No exceptions - chapter overlap verification takes longer than sequential execution.

### Name Files Descriptively  
Date-based filenames cause collisions during archive processing. Use descriptive names like `20260404_animated_brand_compile.txt` instead of generic date patterns.

### Verify Before Fixing
Encoding bugs visible only in PowerShell display are not actual bugs. Signal interrupts during Claude Code are environment artifacts. Always verify problems exist in execution context that matters before attempting fixes.

### Event-Driven Over Scheduled
Only perform work when something happens (file in incoming\). Scheduled execution wastes resources on empty runs. Event-driven scales to zero cost when idle.

### Separate Books Get Separate Files
Cross-project sessions require separate compile outputs per target book. Contamination prevention at file creation beats detection at compile time.

### Pre-Route Content Explicitly
Determine chapter routing during compile session response, not via post-creation auto-detection. Routing maps with human confirmation prevent misrouting errors.

### Storage vs Synthesis Distinction
Session transcripts are ingredients, chapters are meals. Synthesis creates refined knowledge that becomes more valuable over time. Storage preserves raw material that becomes harder to search.

### Output Tokens Dominate API Cost
At 5x input token price, output generation drives expenses. A verbose chapter costs significantly more than concise output from lengthy source material.
